# EchoChain

## Day 2 - Architecture Planning

### Objective

Understand how the Medallion Architecture will be implemented in the EchoChain project using Databricks.

---

## Planned Architecture

The project will use three logical layers:

### Bronze Layer

- Purpose: Store raw data received from the data source.
- Status: Planned

### Silver Layer

- Purpose: Store cleaned and validated data.
- Status: Planned

### Gold Layer

- Purpose: Store analytics-ready datasets for reporting.
- Status: Planned

---

## Notes

The current Databricks Free Edition workspace does not allow creating the required schemas from the Catalog UI. The actual implementation of these layers will begin once the project dataset is available.
